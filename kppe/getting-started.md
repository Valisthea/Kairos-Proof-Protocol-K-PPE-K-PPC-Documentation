# Getting Started

## Prerequisites

- Node.js 20+
- npm or yarn
- A Supabase project (free tier works for development)
- A Railway account (for deployment)

## Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/YOUR_USERNAME/kairos-proof-protocol-engine.git
cd kairos-proof-protocol-engine

# 2. Install dependencies
npm install

# 3. Generate cryptographic keys
npm run generate-keys
# → This outputs environment variables to copy into .env

# 4. Configure environment
cp .env.example .env
# → Paste the generated keys + add Supabase credentials

# 5. Run database migrations
npm run migrate

# 6. Start development server
npm run dev
# → Server running on http://localhost:3000

# 7. Verify installation
curl http://localhost:3000/health
curl http://localhost:3000/health/crypto
```

## Environment Variables

```env
# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_KEY=eyJ...

# Railway
RAILWAY_PORT=3000
NODE_ENV=development

# JWT Signing Keys (generated by generate-keys script)
JWT_ED25519_PRIVATE_KEY=base64...
JWT_ED25519_PUBLIC_KEY=base64...
JWT_PQ_PRIVATE_KEY=base64...
JWT_PQ_PUBLIC_KEY=base64...

# API Secrets
API_HMAC_SECRET=hex...
API_KMAC_SECRET=hex...

# Proof Signing Keys
PROOF_ED25519_PRIVATE_KEY=base64...
PROOF_SPHINCS_PRIVATE_KEY=base64...

# Encryption
ENCRYPTION_MASTER_KEY=hex...

# Redis (optional for dev, required for production)
REDIS_URL=redis://...
```

> ⚠️ **Never commit real keys to the repository.** The `.env` file is in `.gitignore`.

## First API Call

```bash
# 1. Get a SIWE challenge
curl -X POST http://localhost:3000/auth/challenge \
  -H "Content-Type: application/json" \
  -d '{"walletAddress": "0x742d35Cc6634C0532925a3b844Bc9e7595f2bD18", "chainId": 56}'

# 2. Sign the message with your wallet (programmatically or via dApp)
# 3. Verify and get JWT
curl -X POST http://localhost:3000/auth/verify \
  -H "Content-Type: application/json" \
  -d '{"message": "...", "signature": "0x..."}'

# 4. Submit events for proof generation
curl -X POST http://localhost:3000/proofs/batch \
  -H "Authorization: Bearer YOUR_JWT" \
  -H "X-KPE-Timestamp: $(date +%s)" \
  -H "X-KPE-Nonce: $(openssl rand -hex 16)" \
  -H "X-KPE-Sig-Classic: ..." \
  -H "X-KPE-Sig-PQ: ..." \
  -H "Content-Type: application/json" \
  -d '{"events": [...]}'

# 5. Verify a proof (no auth required!)
curl http://localhost:3000/proofs/verify/BATCH_ID/EVENT_HASH
```

# API Reference

## Base URL

```
Development: http://localhost:3000
Production:  https://kpe.kairos-analytics.io  (Railway)
```

## Authentication

All protected endpoints require:
1. `Authorization: Bearer <KPP-HYBRID-1 JWT>` header
2. `X-KPE-Timestamp`, `X-KPE-Nonce`, `X-KPE-Sig-Classic`, `X-KPE-Sig-PQ` headers

Public endpoints (health, proof verification) require no authentication.

## Response Format

All responses are JSON:

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "pq_status": "SIMULATED",
    "protocol_version": "0.1.0",
    "timestamp": 1708776000
  }
}
```

Error responses:

```json
{
  "success": false,
  "error": {
    "code": "AUTH_DUAL_SIG_FAILED",
    "message": "Classical signature valid, post-quantum signature invalid",
    "details": {
      "classical_valid": true,
      "pq_valid": false
    }
  }
}
```

## Endpoints Overview

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/auth/challenge` | No | Get SIWE message to sign |
| `POST` | `/auth/verify` | No | Verify signature, get JWT |
| `POST` | `/auth/refresh` | JWT | Refresh access token |
| `POST` | `/proofs/batch` | JWT + HMAC | Submit events for proof generation |
| `GET` | `/proofs/:batchId` | JWT | Get a proof batch |
| `GET` | `/proofs/verify/:batchId/:eventHash` | **No** | Public proof verification |
| `GET` | `/analytics/events` | JWT | Get analytics data (future) |
| `GET` | `/health` | No | Service health |
| `GET` | `/health/crypto` | No | Crypto module self-test |

See individual endpoint pages for detailed request/response documentation.
